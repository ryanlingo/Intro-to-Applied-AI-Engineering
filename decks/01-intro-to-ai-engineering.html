Intro to Applied AI Engineering

The craft of building reliable systems around AI — not just better prompts.

before we start, a warning

The trap: AI as an answer machine.

Most people meet AI through a chatbot — type a question, get an answer. Handy, but it quietly trains you to treat AI as a vending machine for facts.

The rest of this deck is about everything that view leaves out.

first, the big picture

What is AI engineering?

The craft of designing the setup around an AI model so it reliably does useful work — not fishing for the one clever prompt that happens to land.

the plan

Where we're headed.

Four levels, how they land on an LLM, how to apply them — ending with a real agent you can run.

why it's even needed

An LLM is powerful, but blind.

It knows only two things: the patterns it soaked up in training, and whatever you put in front of it right now. Give it nothing else and it will guess — confidently.

the whole idea on one screen

Each level is made of the one below it.

Start with one stored answer. Find the rule across many. Turn the rule into an exact model. Then build something that makes new models on its own — each step abstracts further and reaches problems the one below it could not.

the split that runs through everything

Two kinds of intelligence.

crystallized skill

A library of procedures for problems you've already met. Fast and reliable — until the problem is genuinely new. A road network: fixed routes between fixed places.

fluid intelligence

Building a new procedure for a problem you've never seen — from just a few examples. A road-building company: lay a road wherever you need to go.

Skill is not intelligence. On the spectrum, the first three rungs are skill — only the top, the meta-model, is the fluid kind.

the same split, in your work

Exploit vs. explore.

exploit what you know

There's a known procedure. Map it into steps and hand the routine ones to AI — crystallized skill is enough.

explore the unknown

There's no procedure to follow. You and the AI have to invent one — this is where the fluid kind earns its keep.

Match the task to the kind of intelligence it needs — and most of the value is in the second.

phase 1

Factoid

A stored answer, no variable.

The bottom of the spectrum: a single memorized point. 1 + 1 = 2, kept as a bare answer, tells you nothing about 2 + 3. It generalizes to nothing — store a million and you're still stuck the moment a question doesn't match one you kept.

Degree zero of abstraction — pure memory, no reach.

phase 2

Organized knowledge

A rule fit to examples.

Gather many factoids and a pattern appears: you learn addition as a rule with a variable, a + b. It reaches beyond what you saw — but only roughly. Solid on familiar sums, shaky on huge or unusual ones. It's how an LLM "adds," and why it slips.

Weak abstraction — interpolation between cases you've seen.

phase 3

Model

An exact, causal rule.

A model captures how the answer is actually produced — not a pattern guessed from examples. Addition as an algorithm: line up the digits, carry the 1. It's right for any two numbers, however large, seen before or not — correct 100% of the time.

Strong abstraction — concise, causal, exact.

phase 4

Meta-model

It builds new models.

The top of the spectrum doesn't apply a model — it makes one for a problem it was never prepared for. Show it two examples of a brand-new operation and it infers the rule itself, from almost nothing. A road network gets you between fixed points; this is the road-building company.

Generating abstractions on the fly — this is intelligence.

now bring in the LLM

What is an LLM, in these terms?

It was trained on oceans of factoids and squeezed them into organized knowledge — patterns it applies to things it never saw.

the four levels, through an LLM

You climb by what you add around it.

Factoid→write a prompt — a single, direct ask Organized knowledge→assemble its context — the organized body of facts, files, and examples it draws on Model→wire a workflow — a fixed, reliable procedure Meta-model→let it run as an agent — it builds its own procedure

Same model every time — the rung changes with how much structure you wrap around it.

the heart of the craft

It's all about context.

Context is everything you put in front of the model so it can do the job — its goal, the facts and files that matter, examples of good work, the tools it can use, the skills it can apply, and the limits.

AI engineering is mostly the craft of building the best context.

the top rung, up close

What "agent" means

An agent runs an agent loop. Hand the LLM a goal and some tools; it plans a step, takes it — reads a file, edits code, runs a command — looks at what happened, and loops until it's done.

A meta-model at work: it writes its own procedures, then runs them.

how do you know it's working?

Evaluation.

Not a vibe, and not a generic leaderboard. Read your own outputs, name the specific ways they fail, turn those failure modes into repeatable tests — then fix and re-run.

Look at your data.

to wrap up

From answer machine to engineer.

You no longer just ask for answers — you build the setup the model works inside: prompt → context → workflow → agent, and you judge what comes back.

Whatever the rung, three things stay yours: the goal, the standards, and the limits.

That's the shift: not a better answer, but a system you can trust.

for reference

Glossary.

AImachines doing things that used to need human thinking.
LLMlarge language model — the text-prediction engine behind today's AI.
modelthe trained network itself; an LLM is one kind.
promptthe instruction you give the model for one task.
contexteverything you put in front of the model to do the job.
tokenthe chunk of text a model reads and writes (≈ a word-piece).
toolan action the model can call: search, run code, edit a file.
skilla reusable rule or know-how the model applies.
workflowa fixed chain of steps run in order.
agenta model that plans its own steps and loops until a goal is met.
commanda single instruction you run in the terminal.
evaluationa test that scores whether outputs are good (an "eval").
hallucinationa confident answer that's simply wrong.
opencodethe open-source terminal tool we use to build agents.
